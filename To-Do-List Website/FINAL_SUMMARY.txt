🔐 BEARER TOKEN INFORMATION

Token Format:
  • Length: 64 hexadecimal characters
  • Algorithm: Random byte generation (SHA-256 strength)
  • Generation: bin2hex(random_bytes(32))
  • Storage: MySQL sessions table

Token Lifecycle:
  • Created: Upon successful login
  • Expiration: 7 days (604800 seconds)
  • Renewal: Login again to get fresh token
  • Revocation: Delete from sessions table

Usage in API Requests:
  curl -H "Authorization: Bearer {your_token_here}" \
       -X POST http://localhost/To-Do-List Website/api.php?action=get_logs

💾 DATABASE SCHEMA

users:
  ├─ id (INT, PRIMARY KEY, AUTO_INCREMENT)
  ├─ username (VARCHAR 255, UNIQUE, NOT NULL)
  ├─ password (VARCHAR 255, NOT NULL) [bcrypt hashed]
  └─ created_at (TIMESTAMP)

todos:
  ├─ id (INT, PRIMARY KEY)
  ├─ user_id (INT, FOREIGN KEY → users.id)
  ├─ title (VARCHAR 255, NOT NULL)
  ├─ description (TEXT)
  ├─ completed (TINYINT, default 0)
  ├─ category (VARCHAR 255)
  ├─ priority (VARCHAR 50)
  ├─ due_date (DATE)
  ├─ created_at (TIMESTAMP)
  └─ updated_at (TIMESTAMP)

activity_logs:
  ├─ id (INT, PRIMARY KEY)
  ├─ user_id (INT, FOREIGN KEY → users.id)
  ├─ log_date (DATE, NOT NULL)
  ├─ log_time (TIME, NOT NULL)
  ├─ category (VARCHAR 255, NOT NULL)
  └─ created_at (TIMESTAMP)

sessions:
  ├─ id (INT, PRIMARY KEY)
  ├─ user_id (INT, FOREIGN KEY → users.id)
  ├─ token (VARCHAR 255, UNIQUE)
  ├─ created_at (TIMESTAMP)
  └─ expires_at (TIMESTAMP)

🐛 TROUBLESHOOTING

Issue: "Database Connection Failed"
Fix:
  1. Start MySQL service (Services > MySQL80 > Start)
  2. Verify credentials in config.php
  3. Ensure todo_list database exists
  4. Check MySQL is listening on localhost:3306

Issue: "Invalid or expired token"
Fix:
  1. Login again to get fresh token
  2. Verify token hasn't expired (7 days)
  3. Check header format: "Authorization: Bearer {token}"
  4. Ensure no extra spaces or line breaks

Issue: "Todo tidak ditemukan" or data not persisting
Fix:
  1. Verify tables were created (check phpMyAdmin)
  2. Run insert_test_accounts.sql for test data
  3. Check user_id in todos table matches logged-in user
  4. Review api_errors.log for details

Issue: 404 Not Found
Fix:
  1. Verify file path in htdocs folder
  2. Check URL: http://localhost/To-Do-List Website/filename.html
  3. Restart Apache if needed
  4. Check file permissions (readable)
