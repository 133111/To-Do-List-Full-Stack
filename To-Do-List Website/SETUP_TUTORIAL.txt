## 📋 OPTION 2: MIGRATION (EXISTING DATABASE)

Jika Anda sudah memiliki data yang ingin disimpan:

### Step 1: Backup Database
```
Sangat PENTING! Backup dulu sebelum modifikasi.

Di phpmyadmin:
1. Select database "todo_list"
2. Click "Export"
3. Click "Go"
4. Download file backup.sql
5. Simpan di tempat aman
```

### Step 2: Buat Table Categories
```
1. Buka phpmyadmin
2. Select database "todo_list"
3. Click tab "SQL"
4. Copy-paste code berikut:

CREATE TABLE IF NOT EXISTS categories (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,
    name VARCHAR(255) NOT NULL,
    emoji VARCHAR(10) DEFAULT '📝',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    UNIQUE KEY unique_category_per_user (user_id, name)
);

5. Click "Go"
6. Tunggu sampai selesai
```

### Step 3: Modify Table Todos
```
1. Di phpmyadmin, masih di tab "SQL"
2. Copy-paste code berikut:

ALTER TABLE todos ADD COLUMN category_id INT AFTER user_id;
ALTER TABLE todos ADD FOREIGN KEY (category_id) REFERENCES categories(id) ON DELETE CASCADE;

3. Click "Go"
4. Tunggu sampai selesai
```

### Step 4: Add Indexes
```
1. Di phpmyadmin, masih di tab "SQL"
2. Copy-paste code berikut:

CREATE INDEX idx_user_categories ON categories(user_id);
CREATE INDEX idx_category_todos ON todos(category_id);

3. Click "Go"
4. Selesai!
```

### Step 5: Verifikasi
```
1. Di phpmyadmin, select database "todo_list"
2. Klik table "categories"
3. Pastikan sudah ada (akan terlihat kosong, tapi itu normal)
4. Klik table "todos"
5. Lihat struktur: harus ada column "category_id"
```

## 🐛 COMMON ISSUES & SOLUTIONS

### Error: "Unknown column 'category_id' in 'on duplicate key update'"
**Cause:** Column belum ditambahkan ke table todos  
**Solution:**
```
1. Go to phpmyadmin
2. Select database todo_list
3. Click table todos → Structure
4. Cari column "category_id"
5. Jika tidak ada, jalankan:
   ALTER TABLE todos ADD COLUMN category_id INT AFTER user_id;
```

### Error: "Cannot add or update a child row"
**Cause:** Foreign key constraint violation  
**Solution:**
```
1. Pastikan table categories sudah exist
2. Pastikan FK sudah dibuat dengan benar
3. Jika ada data lama, cleanup:
   DELETE FROM todos WHERE category_id IS NULL AND id NOT IN (SELECT id FROM todos WHERE category_id > 0);
```

### Kategori tidak muncul setelah dibuat
**Cause:** Database tidak ter-update atau cache  
**Solution:**
```
1. Hard refresh browser: Ctrl + Shift + R
2. Clear cache: Ctrl + Shift + Delete
3. Check phpmyadmin jika data ada di table
4. Check browser console (F12) untuk error
```

### Modal tidak bisa dibuka
**Cause:** JavaScript error  
**Solution:**
```
1. Open browser console: F12
2. Lihat error message
3. Pastikan file index.html sudah ter-update
4. Check apakah ada syntax error
```
