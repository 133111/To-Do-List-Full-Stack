                          📊 DATABASE CHANGES
═══════════════════════════════════════════════════════════════════════════════

NEW TABLE: categories
┌─────────────────────────────────────────────────────────────────────────────┐
│ Column        │ Type         │ Details                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│ id            │ INT PK AI    │ Primary Key, Auto Increment                 │
│ user_id       │ INT FK       │ Foreign Key to users.id (ON DELETE CASCADE) │
│ name          │ VARCHAR(255) │ Category name (UNIQUE per user)             │
│ emoji         │ VARCHAR(10)  │ Category icon/emoji (Default: 📝)           │
│ created_at    │ TIMESTAMP    │ Auto: CURRENT_TIMESTAMP                     │
│ updated_at    │ TIMESTAMP    │ Auto update on change                       │
└─────────────────────────────────────────────────────────────────────────────┘

MODIFIED TABLE: todos
  - REMOVED: category (VARCHAR field)
  - ADDED: category_id (INT FK to categories.id)
  - This enables proper relationship management

INDEXES ADDED:
  ✅ idx_user_categories - Optimize user category queries
  ✅ idx_category_todos   - Optimize task by category queries

FRONTEND ARCHITECTURE:
─────────────────────
1. Main Flow:
   Load → Check Auth → Load Categories → Render UI → Listen Events

2. Category Management Functions:
   ┌─ loadCategories()
   │  ├─ Fetch from API
   │  └─ Update global state
   │
   ├─ renderCategories()
   │  ├─ Generate DOM elements
   │  ├─ Attach event listeners
   │  └─ Load tasks for each category
   │
   ├─ openAddCategoryModal()
   ├─ openEditCategoryModal()
   ├─ closeCategoryModal()
   ├─ saveCategory()
   │  └─ Create or Update based on editingCategoryId
   │
   └─ deleteCategory()
      └─ Confirm → Delete from DB → Reload

3. Task Management (Updated):
   - addTask(categoryId) - Now requires category_id
   - loadTasksForCategory(categoryId) - Fetch only for category
   - toggleComplete() - Same as before
   - deleteTask() - Same as before

BACKEND ARCHITECTURE:
────────────────────
1. New PHP Functions:
   addCategory($conn, $input, $userId)
   ├─ Validate input
   ├─ Check duplicate name
   ├─ Insert to categories table
   └─ Return category_id

   getCategories($conn, $userId)
   ├─ Query all user categories
   ├─ Order by created_at
   └─ Return as JSON array

   updateCategory($conn, $input, $userId)
   ├─ Validate input
   ├─ Check duplicate (excluding current)
   └─ Update record

   deleteCategory($conn, $input, $userId)
   ├─ Delete related todos (CASCADE)
   ├─ Delete category
   └─ Return success

2. Modified Functions:
   addTodo() - Added category_id validation
   getTodos() - Added category_id filtering
   Both maintain security & validation
